What is a Design System?

A collection of reusable components, guided by clear standards, that can be assembled together to build any number of applications.

A Design System is NOT...

  • Just a component library
  • Just a style guide
  • Just a Figma file

It IS the full ecosystem

  • Design principles
  • Design tokens
  • Documentation
  • Components & tooling

Design Systems in the Wild

Open-source design systems used by thousands of projects

Material UI (Google) Primer (GitHub) Ant Design (Alibaba)

Why Design Systems?

1

Consistency

Every app looks and feels the same

2

Efficiency

Build once, use everywhere

3

Scalability

10 teams, 50 apps — still consistent

Naming Convention (BEM)


/* Block — the component itself */
.btn { ... }

/* Modifier — a variant of the block */
.btn--primary { ... }
.btn--secondary { ... }

/* State — CSS pseudo-class */
.btn:hover { ... }
.btn:disabled { ... }

Button React Component

Open: src/components/Button.jsx — a thin wrapper around button.css 


import '../design-system/css/button.css'

function Button({ variant, size = 'md',
                   children, onClick, disabled }) {
  const className = `btn btn--${variant} btn--${size}`

  return (
    <button className={className}
            onClick={onClick} disabled={disabled}>
      {children}
    </button>
  )
}
CSS does ALL the styling.
React only applies the right classes.
The component is a thin wrapper.

Design Tokens

Tokens are named design decisions.
#2563eb → "primary color"
8px → "small spacing"
4px → "small border radius"

A token is a name for a value.
Change the value in one place — it updates everywhere.

Token Categories

Color
--color-primary, --color-surface, --color-text
Spacing
--spacing-sm, --spacing-md, --spacing-lg
Typography
--font-size-sm, --font-size-md, --font-size-lg
Radius
--radius-sm, --radius-md, --radius-lg
Shadows
--shadow-sm, --shadow-md, --shadow-lg

Token Hierarchy

Global Tokens

(raw values)
  • --blue-500: #2563eb
  • --blue-600: #1d4ed8
  • --gray-100: #f3f4f6
  • --gray-800: #1f2937
  • --space-4: 0.25rem
  • --space-8: 0.5rem
  • --radius-4: 4px
  • --radius-8: 8px

Semantic Tokens

(intent / meaning)
  • --color-primary
  • --color-primary-hover
  • --color-surface
  • --color-text
  • --spacing-sm
  • --spacing-md
  • --radius-sm
  • --radius-md

Component Tokens

(specific usage)
  • --btn-bg-primary
  • --btn-bg-hover
  • --card-bg
  • --btn-text, --card-text
  • --btn-padding-y
  • --btn-padding-x
  • --btn-radius
  • --card-radius

Global → Semantic → Component — each level adds meaning

CSS Custom Properties

:root {

.btn--primary {

Typography React Component

Open: src/components/Typography.jsx 


import '../design-system/css/typography.css'

const tagMap = {
  h1: 'h1', h2: 'h2', h3: 'h3',
  body: 'p', caption: 'span'
}

function Typography({ variant = 'body', align = 'left', children }) {
  const Tag = tagMap[variant]

  return (
    <Tag
      className={`typo typo--${variant}`}
      style={{ textAlign: align }}
    >
      {children}
    </Tag>
  )
}

Props: variant ('h1'|'h2'|'h3'|'body'|'caption'), align, children

How Theming Works

CSS custom properties + data attributes 


:root, [data-theme="light"] {
  --color-bg:      #ffffff;
  --color-surface: #f3f4f6;
  --color-text:    #1f2937;
}
              

[data-theme="dark"] {
  --color-bg:      #111827;
  --color-surface: #1f2937;
  --color-text:    #f9fafb;
}
              

<html data-theme="light">  <!-- light tokens -->
<html data-theme="dark">   <!-- dark tokens  -->
Components do not change. Only tokens change.
